## 🧠 The Problem We're Solving

> *"Every Shopify merchant knows how to rank on Google. Nobody knows how to rank in ChatGPT."*

AI shopping agents are rapidly displacing traditional search for a growing segment of high-intent buyers. Unlike Google's crawlers — which rank URLs by backlinks and keyword density — **AI agents extract semantic meaning**. They need to understand *what* a product is, *who* it's for, *what* it's made of, and *why* someone should trust the store.

A Shopify store perfectly optimized for SEO keywords like `"best winter jacket 2024"` may rank #1 on Google, but **completely disappear** when an AI agent is asked:

> *"What jacket is best for hiking in -10°C that packs down small and is waterproof?"*

The gap exists because:

| Traditional SEO | AI Agent Optimization |
|---|---|
| Keyword density | Semantic entity coverage |
| Backlinks | Trustworthiness signals |
| Page speed | Spec completeness |
| Meta descriptions | Natural language context |
| Title tags | Intent-matching descriptions |

**There is no tool that tells merchants how AI agents see their store. We built one — in a weekend.**

## 🚀 What It Does

AI Store Optimizer is a **zero-install SaaS platform** — just enter a Shopify store URL and get a complete AI readiness audit in seconds. No OAuth. No app installation. No credentials required.

---

### 🔬 1. Core Diagnostic Engine — AI Readiness Score

Fetches your store's public product catalog and runs it through **Groq's Llama 3.3 70B** to simulate how AI shopping agents perceive your store.

```
Store: mountaingear.myshopify.com

AI Readiness Score: 61 / 100  ⚠️ Needs Improvement

Critical Gaps Found:
  ✗ 8/12 products missing material composition
  ✗ 5/12 products lack use-case targeting
  ✗ 0/12 products mention return policy inline
  ✗ Brand tone detected: ambiguous (neutral/generic)

Top Strength:
  ✓ Price anchoring present in 9/12 products
```

| Output | Description |
|---|---|
| **AI Readiness Score** (0–100) | Quantified measure of AI discoverability |
| **Critical Gaps** | Missing specs, weak descriptions, absent policies — ranked by impact |
| **AI-Generated Fixes** | Exact copy-paste text to resolve each gap |
| **Semantic Entity Extraction** | How LLMs classify your brand tone, intent, and entities |
| **AI Readiness Trend** | Score trajectory visualization over time |

---

### 🛠️ 2. Product Auto-Fix Hub

Select any product → AI rewrites its title and description with rich semantic keywords designed specifically for NLP parsers. One-click **"Push to Live Store"** action (simulated in demo; real write access via OAuth in v2.0).

**Before:**
> *Alpine Pro X — A great jacket for the outdoors.*

**After (AI-Optimized):**
> *Alpine Pro X Waterproof Shell Jacket — Designed for alpine hiking and cold-weather mountaineering. 3-layer Gore-Tex construction, packable to 1.2L, rated to -15°C. Ideal for intermediate to advanced hikers who prioritize warmth-to-weight ratio.*

## 🛠️ Tech Stack

| Layer | Technology | Why We Chose It |
|---|---|---|
| **Framework** | Next.js 16 (App Router) | Server components + API routes in one codebase — no separate backend |
| **Language** | TypeScript 5 | End-to-end type safety; caught dozens of bugs during rapid development |
| **Styling** | Vanilla CSS | Custom glassmorphic design system, zero runtime overhead |
| **Charts** | Recharts | Lightweight, composable React charts for score trend visualization |
| **State** | React Context + localStorage | Zero-dependency session persistence across pages |
| **AI Model** | Llama 3.3 70B via Groq | Sub-second inference at ~500 tok/s — the only AI fast enough for real-time UX |
| **Data Source** | Shopify `/products.json` | Public API on every Shopify store — zero-install by design |
| **Hosting** | Vercel | Auto-deploy on push, global edge network, free tier for demos |

### ⚡ Why Groq over OpenAI?

Groq's custom LPU (Language Processing Unit) hardware delivers **~500 tokens/second** — making AI analysis feel instantaneous rather than like a loading screen. OpenAI GPT-4o averages 50–80 tok/s. For a real-time diagnostic tool where users are watching results stream live, speed isn't a nice-to-have — it's the product.

| Provider | Speed | Latency | Cost (1M tok) |
|---|---|---|---|
| **Groq (Llama 3.3 70B)** | ~500 tok/s | ~200ms | ~$0.59 |
| OpenAI GPT-4o | ~60 tok/s | ~800ms | ~$5.00 |
| Anthropic Claude 3.5 | ~80 tok/s | ~600ms | ~$3.00 |

### 🔓 Why Zero-Install Architecture?

Traditional Shopify apps require OAuth, app store review, admin credentials, and merchant trust. Our architecture uses the **public `/products.json` endpoint** that every Shopify store exposes by default — meaning any judge, merchant, or investor can test any live store in under 10 seconds, with nothing installed.

This also means:
- **No onboarding friction** — URL → insights in one step
- **Works on competitor stores** — enabling the competitive intelligence feature
- **No security surface** — we never touch merchant credentials

## ⚡ Quick Start

```bash
# Clone the repository
git clone https://github.com/Mridul-gupta678/AI_store_optimizer.git
cd AI_store_optimizer

# Install dependencies
npm install

# Configure environment (only ONE key needed)
```

## 🔌 API Reference

| Endpoint | Method | Payload | Returns |
|---|---|---|---|
| `/api/analyze` | `POST` | `{ domain }` | AI Score (0–100), Gaps, Strengths, Brand Perception |
| `/api/system` | `POST` | `{ domain }` | Store metadata, product count, engine status |
| `/api/products` | `POST` | `{ domain }` | Product catalog `[{ id, title, descriptionHtml }]` |
| `/api/optimize` | `POST` | `{ title, description }` | AI-rewritten title & description |
| `/api/simulate-search` | `POST` | `{ query, products }` | Verdict, confidence %, reasoning log |
| `/api/benchmark` | `POST` | `{ myDomain, competitorDomain }` | Scores, competitor weaknesses, action plan |
| `/api/products/update` | `POST` | `{ id, descriptionHtml }` | Simulated push-to-live (1.5s delay) |
| `/api/themes/inject` | `POST` | `{ domain }` | JSON-LD schema injection simulation |

## 🛡️ Resilience & Error Handling

The platform is engineered to **never crash during a demo**, regardless of network conditions or API state:

| Failure Scenario | Protection Mechanism |
|---|---|
| Store blocked by Cloudflare WAF | `AbortSignal.timeout(3500ms)` + realistic mock product fallback |
| Groq API rate limit (429) | Full mock analysis JSON — UI renders normally, user never sees an error |
| LLM returns malformed JSON | Try-catch parse + automatic markdown fence stripping |
| Invalid or non-Shopify URL entered | Regex sanitization strips `http://`, `www.`, trailing `/` |
| Missing API keys | Descriptive error states, graceful UI degradation |
| Network timeout | Automatic retry with exponential backoff |
